| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-22 | Information Disclosure | Enumeração de e-mails no pedido de reset | mitigate | Resposta única "Se o e-mail existir..." em todos os ramos |
| T-1-23 | Tampering | Open redirect via parâmetro next | mitigate | next validado como path interno; teste unitário dedicado (Teste 3) |
| T-1-24 | Spoofing | Reuso/forja de link de recovery | mitigate | Fluxo PKCE nativo (token_hash single-use, expiração 1h configurada no 01-02); nada hand-rolled |
| T-1-25 | Elevation of Privilege | Senha fraca na redefinição | mitigate | Mínimo 8 no cliente + política do Supabase Auth no servidor (config.toml do 01-02); proteção HaveIBeenPwned fica para o plano Pro (Assumption A3) |
</threat_model>
