## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Gestor → Admin API do GoTrue | Remoção usa service_role após checagem de papel |
| Super-admin → dados de qualquer org | Travessia de RLS legítima, mas auditável |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-26 | Broken Access Control | Sessão do removido continua viva | mitigate | RLS por lookup (efeito imediato) + signOut global + ban (A5); teste 2 prova sem refresh |
| T-1-27 | Elevation of Privilege | Gestor remove membro de outra org / advogado remove alguém | mitigate | Checagem de papel + mesma-org no handler; 404 cross-org (sem vazamento de existência); teste 3 |
| T-1-28 | Repudiation | Super-admin opera sem rastro (backdoor silencioso) | mitigate | logAudit obrigatório em org.created, gestor.invited, member.removed; audit_log legível só por super_admin |
| T-1-29 | Denial of Service | Org fica sem nenhum gestor ativo | mitigate | Handler bloqueia remoção do último gestor ativo |
| T-1-30 | Tampering | Dados históricos apagados na remoção | mitigate | Soft-removal apenas (D-11); nenhum delete em profiles/auth.users no código do endpoint |
</threat_model>
