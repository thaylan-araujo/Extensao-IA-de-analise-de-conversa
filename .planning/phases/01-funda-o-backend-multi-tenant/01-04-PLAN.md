## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → proxy.ts → Server Components | Sessão por cookies httpOnly; claims de UI |
| Painel → Postgres | Queries do painel rodam com o JWT do usuário (RLS) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-12 | Spoofing | Sessão forjada/expirada no painel | mitigate | @supabase/ssr com cookies httpOnly; getUser() no proxy valida contra o servidor de auth a cada request |
| T-1-13 | Elevation of Privilege | Advogado acessa dados do painel burlando o gate de UI | mitigate | Gate de UI (decideRedirect) é conveniência; a barreira real é RLS — advogado só lê o próprio profile/conversas mesmo se atingir a rota (testado no 01-03) |
| T-1-14 | Information Disclosure | Mensagem de login revela existência de conta | mitigate | Mensagem única "E-mail ou senha inválidos." para qualquer falha |
| T-1-15 | Elevation of Privilege | Cliente admin (service role) importado em componente de cliente | mitigate | admin.ts com guarda server-only; acceptance criteria restringe o uso do env var a admin.ts |
</threat_model>
