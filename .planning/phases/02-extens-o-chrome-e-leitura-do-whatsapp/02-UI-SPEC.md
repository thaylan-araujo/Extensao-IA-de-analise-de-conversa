# Phase 02 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Surface desta fase:** painel lateral injetado no WhatsApp Web via Shadow DOM (novo app `apps/extension`, WXT + React 19 + Tailwind 4). Referências de comportamento: WaSpeed / WaLeads (painel flush na borda direita, aba fina quando recolhido, login dentro do painel).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — shadcn é escopo do dashboard (CLAUDE.md); o painel da extensão vive em Shadow DOM e usa componentes próprios com Tailwind 4 |
| Preset | not applicable |
| Component library | none (componentes próprios em React 19, estilizados com Tailwind 4 escopado dentro do Shadow DOM) |
| Icon library | lucide-react (tree-shakeable; ícones: ChevronLeft/ChevronRight para expandir/recolher, LogOut, AlertTriangle, Circle/status dot) |
| Font | `system-ui, -apple-system, "Segoe UI", Arial, Helvetica, sans-serif` — harmoniza com o WhatsApp Web e com o stack Arial da Fase 1; declarar no `:host` do Shadow DOM (o painel NÃO herda fontes da página) |

**Restrição arquitetural:** todo CSS vive dentro do Shadow DOM (Tailwind injetado no shadow root via WXT `cssInjectionMode: "ui"` ou equivalente). Nenhum estilo pode vazar para o DOM do WhatsApp Web, e nenhum estilo do WhatsApp pode ser assumido dentro do painel — declarar reset básico no `:host`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap ícone-texto, padding do status dot |
| sm | 8px | Espaçamento entre elementos compactos (linhas de metadados, gap label-input) |
| md | 16px | Padding interno do painel e dos cards, gap entre campos de formulário |
| lg | 24px | Separação entre seções do painel (status / teaser IA / rodapé) |
| xl | 32px | Respiro vertical do formulário de login e dos estados vazios |
| 2xl | 48px | Não usado nesta fase (painel compacto) — reservado |
| 3xl | 64px | Não usado nesta fase — reservado |

Exceptions (constantes de layout, todas múltiplas de 4):

- **Altura mínima de toque: 44px** (`h-11`) para botões e inputs — padrão herdado da Fase 1.
- **Largura do painel expandido: 360px**, altura 100vh, colado (flush) na borda direita.
- **Largura da aba recolhida: 40px**, altura 100vh (faixa fina com logo no topo, seta de expandir e status dot sempre visível — D-07/D-08).
- **Status dot: 8px** de diâmetro.

## Typography

Exatamente 4 tamanhos e 2 pesos. Nenhum outro valor é permitido nesta fase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption (horários, metadados, badge "Em breve", rótulo do status) | 12px | 400 | 1.5 |
| Body (texto corrido, avisos, labels de formulário, texto de botão) | 14px | 400 (texto) / 600 (labels e botões) | 1.5 |
| Heading (títulos de seção, inputs a 16px/400 para evitar zoom — padrão `text-base` da Fase 1) | 16px | 600 (headings) / 400 (inputs) | 1.2 (headings) / 1.5 (inputs) |
| Display (título do painel no login e no cabeçalho) | 20px | 600 | 1.2 |

- Pesos permitidos: **400 (regular)** e **600 (semibold)**. Não usar `font-medium` (500) nem `font-bold` (700) no painel.
- Todo texto em português brasileiro. Sem CAPS LOCK decorativo; capitalização de frase ("Sugerir resposta", não "SUGERIR RESPOSTA").

## Color

Valores = tokens da paleta padrão do Tailwind 4 (canônicos para o executor; hex aproximado entre parênteses).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` (#ffffff) | Fundo do painel, formulário de login, área de conteúdo |
| Secondary (30%) | `zinc-50` (#fafafa) superfícies secundárias; `zinc-200` (#e4e4e7) bordas/divisores; `zinc-950` títulos; `zinc-800` texto corpo; `zinc-600` texto secundário/metadados | Cabeçalho do painel, cards de seção, aba recolhida, hierarquia de texto |
| Accent (10%) | `emerald-700` (~#047857); hover `emerald-800`; dot ativo `emerald-500`; focus ring `emerald-100` | Ver lista reservada abaixo |
| Destructive | `red-700` (~#b91c1c) texto; `red-50` fundo de erro | Mensagens de erro de login/validação e confirmação de "Sair" |
| Warning (semântica, exigida por EXT-05/D-13) | `amber-50` fundo; `amber-300` borda; `amber-900` texto; ícone `amber-600` | EXCLUSIVO para: aviso de leitura quebrada/kill-switch (D-13), "grupo não monitorado" (D-02) e "acesso desativado" (D-11) |

Accent reserved for (lista fechada — nada além disto):

1. Botão primário "Entrar" (login)
2. Link "Esqueci minha senha"
3. Status dot verde + rótulo "Conversa monitorada" (D-08) — usa `emerald-500` no dot
4. Focus ring de inputs e botões (`emerald-100` ring + `emerald-600` borda, padrão da Fase 1)
5. Botão "Sugerir resposta" do teaser (D-05) — renderizado DESATIVADO nesta fase: fundo `zinc-100`, texto `zinc-400`, badge "Em breve"; ele só ganha o accent quando a Fase 3 o ativar

O accent NUNCA aparece em: bordas decorativas, cabeçalho do painel, ícones neutros, textos de corpo.

## Layout & Panel Behavior (contrato de interação)

Decisões travadas em CONTEXT.md — o executor não tem liberdade aqui:

| Comportamento | Contrato | Fonte |
|---------------|----------|-------|
| Posição | Painel flush na borda direita, 100vh. O WhatsApp Web é comprimido (largura reduzida), nunca sobreposto — nenhuma mensagem do chat pode ficar coberta | D-07, EXT-01 |
| Recolhido | Aba fina de 40px com logo, seta de expandir e status dot sempre visível | D-07, D-08 |
| Estado inicial | Aberto no primeiro uso; depois persiste a última escolha (aberto/recolhido) entre sessões | D-06 |
| Recolher ≠ pausar | Recolher o painel não interrompe a sincronização; o status dot na aba continua refletindo o estado real | D-04, D-08 |
| Transição | Expandir/recolher com transition de `width`/`transform` 200ms ease-out; nenhuma outra animação no painel | default |
| Somente leitura | O painel nunca escreve no DOM do WhatsApp nem insere texto no campo de mensagem | EXT-08, Out of Scope |
| Login | Formulário e-mail/senha dentro do próprio painel (não no popup da extensão) | D-09 |

### Estados do painel (todos precisam de design)

| Estado | Conteúdo |
|--------|----------|
| Deslogado | Título do produto (Display 20px), formulário e-mail/senha, botão "Entrar", link "Esqueci minha senha" |
| Logado + conversa individual ativa | Cabeçalho (logo, e-mail do usuário, botão recolher), card do lead (nome/telefone da conversa ativa), status "Conversa monitorada" com dot verde, seção teaser IA desativada, rodapé com "Sair" |
| Logado + nenhuma conversa aberta | Estado vazio (heading + body do contrato de copy) |
| Logado + grupo/comunidade aberto | Painel desativado com aviso warning "Grupos não são monitorados" (D-02) |
| Leitura quebrada / kill-switch ativo | Banner warning com o texto D-13; controles de IA e status ficam ocultos/desativados; login e painel continuam vivos | 
| Acesso removido (D-11) | Painel troca todo o conteúdo por aviso warning "Seu acesso foi desativado" + orientação; leitura para; controles somem |
| Carregando (login em andamento) | Botão com "Entrando..." e `disabled` (padrão da Fase 1) |

### Teaser da IA (D-05)

Seção "Copiloto de IA" sempre presente quando logado: botão "Sugerir resposta" desativado + badge Caption "Em breve", e um bloco placeholder "Diagnóstico da conversa" desativado com o mesmo badge. Visual: fundo `zinc-50`, texto `zinc-400`, cursor `not-allowed`. A Fase 3/4 apenas ativa — o layout não muda.

## Copywriting Contract

Tom (herdado da Fase 1 / D-13): profissional, direto, tratamento por "você", sem emojis, pt-BR.

| Element | Copy |
|---------|------|
| Primary CTA (login) | "Entrar" — carregando: "Entrando..." |
| Empty state heading (sem conversa aberta) | "Nenhuma conversa aberta" |
| Empty state body | "Abra uma conversa individual no WhatsApp para acompanhar o atendimento." |
| Error state (login inválido) | "E-mail ou senha inválidos." (idêntico à Fase 1) |
| Error state (leitura quebrada / kill-switch — D-13, texto travado) | "O WhatsApp mudou e estamos ajustando a leitura. Suas conversas não estão sendo registradas neste momento e as funções de IA estão pausadas. Você não precisa fazer nada — voltaremos automaticamente." |
| Grupo aberto (D-02) | Heading: "Grupos não são monitorados" — Body: "Abra uma conversa individual para acompanhar o atendimento." |
| Acesso removido (D-11) | "Seu acesso foi desativado — fale com seu gestor." |
| Status de monitoramento (D-08) | Ativo: "Conversa monitorada" (dot `emerald-500`) — Pausado: "Monitoramento pausado" (dot `amber-600`) — Grupo: "Não monitorada" (dot `zinc-400`) |
| Teaser IA (D-05) | Botão: "Sugerir resposta" + badge "Em breve"; bloco: "Diagnóstico da conversa" + badge "Em breve" |
| Link secundário (login) | "Esqueci minha senha" (abre o fluxo web da Fase 1 em nova aba) |
| Destructive confirmation (Sair) | Sair: "Sair da conta? O monitoramento das conversas fica pausado até você entrar novamente." Botões: "Sair" (destructive) / "Cancelar" |

Regras: mensagens de erro sempre dizem o problema + o próximo passo; nenhuma mensagem culpa o usuário; o aviso de quebra afirma explicitamente que a coleta está pausada (transparência exigida pelo dono).

## Acessibilidade (mínimo desta fase)

- Contraste AA: `zinc-800` sobre branco e `amber-900` sobre `amber-50` atendem 4.5:1; nunca usar `zinc-400` para texto informativo (só para estados desativados e dot neutro).
- Foco visível em todos os controles (ring emerald, padrão Fase 1); painel navegável por teclado (Tab; Esc não fecha o painel — recolher é só pelo botão).
- Status dot sempre acompanhado de rótulo textual (nunca cor sozinha comunicando estado).
- `aria-label` em pt-BR nos botões só-ícone (ex.: "Recolher painel", "Expandir painel").
