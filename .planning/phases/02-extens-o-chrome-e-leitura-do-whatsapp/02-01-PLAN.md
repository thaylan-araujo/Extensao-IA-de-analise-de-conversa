## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm → node_modules | Pacotes de terceiros entram no build da extensão (que roda dentro do WhatsApp Web de todo advogado) |
| Página WhatsApp → content script | DOM da página é input não confiável lido pelo coletor de spike |
| Extensão → Chrome Web Store | Manifest amplo = revisão mais lenta + superfície maior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-SC | Tampering | npm installs (apps/extension) | mitigate | Auditoria de legitimidade da pesquisa + checkpoint humano bloqueante (Task 1) + versões exatas pinadas; zero postinstall nos pacotes auditados |
| T-02-01 | Information Disclosure | spike/collector.ts | mitigate | Coletor só loga localmente no console; fixtures são sanitizadas manualmente (nomes/telefones substituídos) antes de entrar no git |
| T-02-02 | Tampering | manifest MV3 | mitigate | Manifest mínimo (permissions só storage, matches só web.whatsapp.com) — Pattern 8; verificado por assertiva automatizada na Task 2 |
| T-02-03 | Elevation of Privilege | coletor no bundle de produção | mitigate | Coletor registrado apenas sob import.meta.env.DEV; build de produção (02-07) não o inclui |
</threat_model>
